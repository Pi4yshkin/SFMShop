Пользователь вводит URL, например sfmshop.com
Клиент проверяет кеш DNS, есть ли там уже этот IP
Если нет, то DNS зарпашивает преобразование URL в IP адресс и сохраняет в кеше (время выполнения 10 - 50 мс)
Проверка в терминале:
dig sfmshop.com
    ;; Query time: 369 msec
    ;; SERVER: 1.1.1.1#53(1.1.1.1)
    ;; WHEN: Fri Apr 03 08:12:37 MSK 2026
    ;; MSG SIZE  rcvd: 56
Возможные проблемы: сервер DNS недоступен

Устанавливается TCP соединение (время выполнения 50 - 200 мс)
Проверка в терминале:
pi4yshkin@MacBook-Air-Aleksandr ~ % ping sfmshop.com
PING sfmshop.com (63.141.128.16): 56 data bytes
Возможные проблемы: сервер недоступен

Если соединение установлено, отправляется HTTP запрос.
Проверка в терминале: 
pi4yshkin@MacBook-Air-Aleksandr ~ % curl -s -o /dev/null -w "HTTP Code: %{http_code}\nTime: %{time_total}s\n" https://sfmshop.com         
HTTP Code: 200
Time: 1.621499s
ps. при вводе curl https://sfmshop.com  выдает просто огромное количество кода, поэтому пришлось погуглить как сделать красивее.
Возможные проблемы: БД недоступна

Обработка запроса на сервере (время выполнения 100 - 500 мс)
Возможен долгий процесс обработки запроса из-за большого размера файла или не оптимизированного кода.

Сервер отправляет ответ клиентелиенту.
